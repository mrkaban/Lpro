Типы лицензий на программы

Каждое программное обеспечение поставляется с лицензионным соглашением, которое обговаривает условия использования. Лицензионное соглашение является документом с юридическим весом, поэтому устанавливая программу Вы автоматически соглашаетесь с условиями лицензионного соглашения, даже если оно не отображалось при установке.

Типы лицензий на программное обеспечение можно подразделить на:
  • Платное программное обеспечение;
  • Условно-бесплатное программное обеспечение;
  • Свободно-распространяемое программное обеспечение.

 Некоторые приписывают свободно-распространяемые программы к категории бесплатных, и в эту же категорию добавляют условно-бесплатные. Это неправильный подход, потому что условно-бесплатные программы можно использовать бесплатно только при жестком соблюдении условий лицензионного соглашения.

Также неправильно причислять платное программное обеспечение к другим категориям. Они все таки требуют оплаты для корректной работы.

Свободно-распространяемые программы это отдельная категория, и причислять её к другим некорректно. Пользователь может использовать их не обращая внимания на лицензионное соглашение, потому что они разрешают использовать и дома, и в офисе. Ограничения имеются только на распространение и модификацию.

Public Domain - В эту категорию относят программное обеспечение, авторское право на которое истекло. Некоторые относят Public Domain (общественное достояние) к свободному программному обеспечению, но это ошибочно, так как данная категория сильно зависит от законодательства страны, в которой используется или была написана программа. К тому же, в таких программах сохраняются старые лицензионные соглашения, в которых не говорится, что программу можно использовать свободно, поэтому в случае проверки могут быть неприятности.

Shareware - Программное обеспечение с возможностью оплаты, которое может предполагать демонстрационный период.

Commercial software (коммерческое программное обеспечение) - Изначально создается с целью коммерческой прибыли, обычно путем продажи экземпляров.

Crippleware - Ключевые функциональные возможности отключаются до приобретения ключа.

Hostageware - Имеет ограничения по времени, или по количеству лицензий, которые можно разблокировать после оплаты.

Nagware - Навязчивые напоминания о необходимости оплаты.

Trialware, Trial, Demoware - Демонстрационная версия коммерческого программного продукта, которая имеет ограничения по времени или функциональности.

Careware, Charityware - Оплата идёт на благотворительность.

Abandonware - Уже не выставляется на продажу и не поддерживается.

Adware - С содержанием рекламы.

Beerware - Разрешает пользователю использовать бесплатно программу и исходный код, но если он встретит автора, то он должен купить ему кружку пива.

Betaware - Бета-версия, на период тестирования может разрешаться использовать бесплатно.

CDware - Распространяется в рекламных целях на компакт-дисках.

Freeware - Может использоваться бесплатно при соблюдении условий лицензионного соглашения, в которых в большинстве случаев в явной форме запрещено использование в организации.

Donateware, Donationware - Предоставляется бесплатно, обычно даже с разрешением применения в организации, но может выскакивать сообщение с просьбой сделать пожертвование автору.

Linkware - Автор просит на своих сайтах указывать ссылку на его сайт.

Liteware - Коммерческий продукт с ограниченным функционалом.

Postcardware, Cardware - Может использоваться бесплатно, но автор просит выслать ему открытку.

Proprietary software (Проприетарное программное обеспечение) - Является частной собственностью, условия обговариваются в лицензионном соглашении.

Registerware - Требуется заполнить анкету.

Requestware - Автор просил сделать что-то за использование программы.

Для пользователя свободно-распространяемые программы одинаковы, потому что всех их разрешается использовать и дома, и в офисе.

Free software - Общий термин свободного программного обеспечения. Но из-за неоднозначности слова Free его путают с бесплатным программным обеспечением, поэтому было рекомендовано перейти к термину Open source software.

GNU GPL - Самый популярная лицензия на свободное программное обеспечение. Разрешается использовать, распространять и модифицировать. При модификации программы, необходимо в дальнейшем распространять продукт по той же лицензии.

BSD - Обычно в таких лицензия всего три пункта. Первый разрешает использовать продукт бесплатно и дома, и в организации. Второй говорит, что даже при модификации должна сохраняться ссылка на автора исходной программы. Третий пункт говорит о том, что никакие гарантии не предоставляются.

MIT - Разрешается использовать в продуктах с закрытым кодом, при условии того, что текст лицензии распространяется вместе с этой программой.

GNU LGPL - Разработана в качестве компромисса между GNU GPL и такими лицензиями как BSD и MIT.

Open source software - Открытое программное обеспечение, то есть программное обеспечение с открытым исходным кодом. При чем, в данном случае мало открыть код для изучения, он должен быть для доработки по свободным лицензиям.

Apache License - Разрешено использовать и распространять. Модификация также разрешена, но с некоторыми незначительными условиями. Используется для продуктов компании Apache Software Foundation.

Mozilla Public License - Лицензия на свободное ПО, которая разработана Mozilla Foundation, и используется для программных продуктов этой компании.

Academic free license - Лицензия на свободное ПО, которая поясняет, что авторские права сохраняются.

GNU AGPL - Изначально создана для веб-приложений, и разрешает использовать, распространять и модифицировать.

Common Public License - Лицензия на свободное ПО, которая разработана IBM.

Eclipse Public License - Лицензия на открытое ПО, которая разработана Eclipse Foundation, и используется для своих программных продуктов.

Eiffel Forum License - Разрешается использовать, распространять и модифицировать. Но авторские права сохраняются и текст данной лицензии обязательно должен включаться в каждое продукт, основанный на данном ПО.
